VLM_Datapipeline

The process of pairing images and text in a Vision Language Model (VLM) and building a vector database is as follows:

Images are divided into 1024x1024 patches to efficiently utilize the GPU. Then, the images and text are encoded into vectors and paired.

These vector pairs are stored in a vector database, and the original metadata is stored in a relational database.

This constructed database is designed to quickly search for similar images and text.

Vectordb_Milvus

An image vector database is a database that stores and manages images in vector format.

Typically, instead of storing images as pixel values that are easy to understand,

image databases store them in vector format to enable efficient similarity search, retrieval, and analysis.
